## Schema Summary

- Table: hospital_visits
- Purpose: store patient visit records for hospitals
- Primary key: id
- Required: patient_id, hospital_id, visit_time
- Nullable: triage_level, diagnosis

## Assumptions

- `id` is generated by the database
- `created_at` defaults to current timestamp
- `updated_at` defaults to current timestamp and updates on row modification
- `patient_id` and `hospital_id` are foreign-key-like fields, but FK constraints are not added unless confirmed

CREATE TABLE hospital_visits (
    id BIGSERIAL PRIMARY KEY,
    patient_id BIGINT NOT NULL,
    hospital_id BIGINT NOT NULL,
    visit_time TIMESTAMP NOT NULL,
    triage_level INTEGER NULL,
    diagnosis TEXT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
